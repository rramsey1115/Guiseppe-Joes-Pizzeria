Guiseppe Joe's Pizzeria
================
This full-stack application was bulit as a project for Nashville Software School's Server-Side curriculum.

For this project I was given specifications of what the pizza shop owner required his pizza order management system to be able to do for his business. I then had to create the ERD, Wireframe, user-stories, project board, and finally the application itself. 

This was a great project where I learned a lot about complex data such as using multiple foreign keys within one table, editing orders containing serval many-to-many relationships, and updating state using React to reflect changes to orders and order forms in real time. 


Features
============
Only logged in employees are able to view the application
Employees are able to:
- View all orders, organized by current and previous, then by date placed
- Edit an order, with side-by-side live updates to the order receipt
- Delete an order
- Create an order
- Order form is a series of questions which dictates which information is required to place the order (order form changes based on delivery, dine-in, take-out, how many pizzas)
- Exit creation of an order at any time
- Complete an order, marking its completion date/time, assigning the driver who delivered, and the tip amount from customer (if any)
